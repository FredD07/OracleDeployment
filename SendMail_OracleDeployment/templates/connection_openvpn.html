Accessing your environment with OpenVPN

To access your environment, a set of OpenVPN certificates has been generated and attached to this mail.
Those OpenVPN certificates are valid for 364 days.

  1. General
    • You must be sure that your firewall is accepting outgoing connections to the following IP Addresses/ports for the Openvpn connection :
      • Subnet : 129.35.171.0/24
      • Ports : 1194 and 1195
      • Protocols : TCP and UDP
    • Each ZIP file contains an individual certificat. It is not allowed (and not possible) to use the same certificat at the same time from 2 differents machines.
    • If ever you are using another VPN Client (ATT, Cisco AnyConnect), please disconnect before starting the OpenVPN Client, then reconnect to the other VPN

  2. Connect using a Windows Workstation
    • Download OpenVPN client from this page (Version >= 2.4)
    • Install the OpenVPN client
    • Unzip one of the attached zip file and copy the *.ovpn file (ex: ProjectName_xxxxx_01.psscopenvpn.ovpn) to #OpenVPNInstallDirectory#\config
    • Start the OpenVPN Client as Administrator.
      This permission is required because OpenVPN needs to add routes on your machine during the connection. If you don't have admin access, please take a look to the following how-to article:
      http://openvpn.se/files/howto/openvpn-howto_run_openvpn_as_nonadmin.html
    • Right-click on the OpenVPN icon in the taskbar and connect. The icon should turn green if everything is fine.

  3. Connect using a Mac Workstation
    • Download and install an OpenVPN client for MacOSX, like Tunnelblick
    • For more detailed information, please refer to this page: https://openvpn.net/index.php/access-server/docs/admin-guides/183-how-to-connect-to-access-server-from-a-mac.html

  4. Connect using a Linux Workstation
    • Install the OpenVPN client using the OS package manager:
      • CentOS/RHEL : yum install openvpn
      • Ubuntu/Debian : apt-get install openvpn
    • Unzip one of the attached zip file. You will get a *.ovpn file (ex: ProjectName_xxxxx_01.psscopenvpn.ovpn)
    • Start the openVPN client:
      • openvpn --config #PATH_TO_CERTIFICATE#/ProjectName_xxxxx_01.psscopenvpn.ovpn
